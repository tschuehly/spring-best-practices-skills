⏺ SSE Notification System Overview

The project uses PostgreSQL LISTEN/NOTIFY as the event transport, with Spring WebFlux Flux streams delivering events to browsers.

1. Slideshow Updates (live photo feed)
  Guest uploads photo → pg_notify → SlideshowNotificationService → Reactor Sink → SSE stream

2. Ranking Game (live multiplayer game events)
  Game action → pg_notify('ranking_game_$sessionId') → per-session listener → SSE stream

Architecture:
  Browser (EventSource / HTMX)
    ↕ HTTP SSE stream
  Spring Controller (Flux<ServerSentEvent>)
    ↕ Reactor Sink
  NotificationService (listener thread)
    ↕ JDBC polling pgConn.getNotifications(500ms)
  PostgreSQL LISTEN/NOTIFY
    ↑ pg_notify() called by business services

The ranking game feature uses three main database tables:

ranking_game_session
Main session table — id, event_id, status (LOBBY/ACTIVE/FINISHED/REVEAL),
timer_duration_seconds, budget_heart/funny/wow, timestamps

ranking_game_participant
Tracks which guests are playing — session_id, event_guest_id, joined_at
Unique constraint: one participant per guest per session

ranking_game_reaction
Records each reaction — participant_id, submission_id, reaction_type (HEART/FUNNY/WOW)
Unique constraint: one reaction per participant per photo per session

Pre Tool Use — Block Before It Happens

Exit 2 = hard block. Claude cannot proceed.

# git-guardrails.sh
DANGEROUS_PATTERNS=(
  "git push" "git reset --hard"
  "git clean -fd" "git branch -D"
  "git checkout \." "git restore \."
)

for pattern in "${DANGEROUS_PATTERNS[@]}"; do
  if echo "$COMMAND" | grep -qE "$pattern"; then
    echo "BLOCKED: '$COMMAND' matches '$pattern'" >&2
    exit 2
  fi
done

Pre Tool Use — Enforce Conventions

TEST_PATTERNS=(
  "gradlew test " "gradlew test$"
  "gradlew.*--tests"
)

for pattern in "${TEST_PATTERNS[@]}"; do
  if echo "$COMMAND" | grep -qE -- "$pattern"; then
    deny "Use the /test skill instead"
  fi
done
